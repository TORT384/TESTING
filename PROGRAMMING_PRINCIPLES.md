# Принципи програмування у проєкті "Таблиці"

У цьому проєкті продемонстровано дотримання базових принципів структурування та оформлення коду.

---

### 1. Separation of Concerns (Розділення відповідальності)
**Опис:** Принцип полягає в тому, щоб розділяти структуру (HTML) та оформлення (CSS).
* **Демонстрація:** Усі стилі для таблиць, кольорові схеми та відступи винесені в окремий блок `<style>`, а не прописані всередині кожного тегу через атрибут `style`. Це полегшує підтримку коду.
* **Посилання на код:** [Стилі в index.html (рядки 8-36)](https://github.com/TORT384/TESTING/blob/main/index.html#L8-L36)

---

### 2. DRY (Don't Repeat Yourself)
**Опис:** Уникнення дублювання коду.
* **Демонстрація:** Замість того, щоб прописувати `background-color` для кожної клітинки окремо, я створив універсальні класи (наприклад, `.red`, `.yellow`, `.lightblue`). Це дозволяє один раз описати колір і використовувати його багаторазово.
* **Посилання на код:** [Класи кольорів (рядки 17-34)](https://github.com/TORT384/TESTING/blob/main/index.html#L17-L34) та їх використання в [Таблиці 8 (рядки 157-161)](https://github.com/TORT384/TESTING/blob/main/index.html#L157-L161)

---

### 3. Semantic HTML (Семантичність)
**Опис:** Використання тегів за їх призначенням для кращої доступності та структурованості.
* **Демонстрація:** Для заголовків таблиць використовуються спеціальні теги `<th>` замість звичайних `<td>`. Також використано тег `<colgroup>` для групування специфічних стовпців.
* **Посилання на код:** [Заголовки Таблиці 7 (рядки 106-108)](https://github.com/TORT384/TESTING/blob/main/index.html#L106-L108) та [Групування стовпців (рядки 154-164)](https://github.com/TORT384/TESTING/blob/main/index.html#L154-L164)

---

### 4. KISS (Keep It Simple, Stupid)
**Опис:** Принцип простоти. Розв'язання задачі найменш складним шляхом.
* **Демонстрація:** Для об'єднання клітинок використано стандартні атрибути `rowspan` та `colspan`. Це просте та надійне рішення для створення складних структур без використання додаткових бібліотек.
* **Посилання на код:** [Використання rowspan/colspan (рядки 64-75)](https://github.com/TORT384/TESTING/blob/main/index.html#L64-L75)
